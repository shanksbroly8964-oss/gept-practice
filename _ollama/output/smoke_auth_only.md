1. **問題理解**  
   `auth.js` 檔案的主要責任是實現 Firebase Google 登入及 Firestore 雲端同步
，管理使用者身份驗證狀態，並提供登入、登出和訪客模式等功能。

2. **建議修改檔案**  
   要添加一個安全的訪客模式開關，需要在 `auth.js` 中增加對 `ALLOW_GUEST` 變
數的控制，並更新相應的 UI 和邏輯。具體來說，可以在 `initFirebase` 函數中檢查
 `ALLOW_GUEST` 的值，若為 `false` 則不顯示訪客登入按鈕。

3. **風險**  
   - 若訪客模式被啟用，可能會導致未經授權的使用者訪問受限資源。
   - 如果訪客模式下的數據管理不當，可能會泄露敏感信息。

4. **驗證方式**  
   - 測試訪客模式是否能正確顯示和隱藏訪客登入按鈕。
   - 確保訪客模式下無法進行受保護的操作（如保存進度、登出等）。
